The Problem

AI coding tools are fast.
That speed is also the risk.

When your AI can write 500 lines in a minute, it's easy to start building before you've answered the hard questions. Three hours later you're refactoring in code review — or worse, starting over.

The team already knows the right process. SpecGantry makes it impossible to skip it.

Speed without direction

AI writes code instantly, but it doesn't know if you're solving the right problem until after the spec review.

Architecture discovered too late

Violations found in code review cost 10× more to fix than violations caught at spec time.

Invisible token spend

Teams run up hundreds of dollars in AI tokens with no visibility into what each feature actually cost.

Incoherent features

Frontend built without the backend spec. AI integration specced but never wired up. Vertical incoherence discovered at integration.

The Solution

Four phases. Hard gates between every one.

No skipping. No "we'll write the spec later." Every phase gate reads the filesystem — the pipeline advances only when the artifact actually exists on disk.

01
Ideation
architecture.md · project-state.yaml
02
Story Spec
story-spec.md per story
03
Build
build-report.yaml · gap.md (if any)
04
Deploy Release
deploy.sh · deploy-artifact.md
05
[N] New work — post-release
Investigate → confirm → build fix or gap.md → deploy
Gates are two-layer: the orchestrator routes on state flags in project-state.yaml; each subagent enforces its own hard gate against the artifact on disk before doing any work.
Key Features

Everything you need to ship AI-assisted features with confidence

Specs Before Code — Enforced

No code is written until a complete story spec exists and has been checked against every architectural guardrail. The gate is hard: Claude Code won't proceed without it. No workarounds, no exceptions.

Learn about the spec gate →

Architecture as a Living Guardrail

Define the architecture once. Every story spec is automatically checked against it. Violations are flagged before a line of code is written.

Story-Based Vertical Slices

Each story owns its complete vertical slice — UI, backend, data, AI integration, and acceptance criteria in one spec. Incoherence between layers is structurally impossible.

Session-Safe Progress

Every phase writes to disk after each question. Network dropped? Context reset? End of day? Pick up exactly where you left off.

Full Token Cost Visibility

Every agent invocation logs tokens used, model selected, and cost. Dashboard shows spend by phase, story, and release — no surprises.

See cost tracking →

Complete YAML-Based State

All progress saved as plain text, git-friendly YAML. Audit trails, version control, diffs — it's all there. Perfect for session resumption across days.

Story Dependency Management

Stories declare dependencies. The pipeline enforces order — a story that depends on another can't be built until its dependency is complete.

Reverse-Engineer Existing Code

Have an existing codebase? SpecGantry analyzes it and generates an architecture and story backlog. Bring structure to legacy projects.
